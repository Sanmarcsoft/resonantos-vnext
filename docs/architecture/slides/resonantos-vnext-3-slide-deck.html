What Works Now vs What Is Still Being Built

The product already has a real shell, host services, memory, provider routing, add-on manifests, and first tool bridges. The open work is mostly hardening, lifecycle, automation, and safety-gated execution.

Capability Map

Working Now

  • Desktop shell + persisted stateLauncher, central workspace, chat rail, settings.
  • Augmentor chat railThreads, branching, pinning, interruption, Markdown replies.
  • Context memory foundationRaw transcript ledger, compact state, context map.
  • Living Archive host serviceSearch, read, intake, review, trusted promotion.
  • Provider fabricCentral routing, MiniMax path, diagnostics, smoke tests.
  • Add-on SDK V0Validated manifests, capability gates, bundled catalog.
  • Delegation workspacesPacket, TASK.md, monitor, Engineer start bridge.

  • Browser V1Chromium screenshot/evidence surface, not live embedded viewport.
  • Obsidian V1Read-only vault bridge, no automatic sync or embedded editing.

  • Provider strategyCost-aware policy recognized; UI not complete.
  • Chat media + inputAttachments, regenerate, and native dictation are incomplete.
  • Product hardeningChunk warning, cross-platform QA, onboarding, release flow.

Roadmap: From Working Foundation To Product-Ready OS

The next sequence should keep safety boundaries intact while turning the existing architecture into a durable, user-facing operating environment.

Roadmap
1

Stabilize The Shell

Make the app launcher and workspace model the default product experience.

  • Home / Apps first screen
  • Center-launched add-ons
  • Collapsible rails
  • Full-screen workspace mode
2

Harden Memory And Archive

Move from preview/review flows toward audited execution and durable provenance.

  • Reorganisation execution
  • Rollback and audit logs
  • Watcher / sync layer
  • Semantic merge and review UX
3

Complete Tool Fabric

Give add-ons and agents one consistent lifecycle, permission, and evidence model.

  • Add-on lifecycle manager
  • Browser click/type tools
  • Obsidian sync path
  • Provider strategy UI
4

Productize And Release

Turn the macOS build into a tested, cross-platform, release-ready product.

  • Engineer repair workflow
  • Long-session memory tests
  • Accessibility and touch QA
  • Windows / Linux validation
!
